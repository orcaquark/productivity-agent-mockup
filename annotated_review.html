Productivity Agent · Design Review v2

Stress-testing the design against real failure modes

Same three screens as before, hardened against accessibility gaps, trust gaps, irreversible mistakes, edge cases, and privacy leaks we hadn't addressed yet. Each section opens with the original design rationale (tagged R1, R2…) from the first review, followed by what changed in this pass — tagged FIXED for something now visible in the mockup, or FLAGGED for something worth spec'ing separately before build (see the deferred list at the bottom).

What changed since v1

  • Contrast: violet text darkened to pass AA against light backgrounds; buttons keep full-saturation violet
  • Touch targets: every tappable element is now ≥40–50px tall, not just visually sized
  • Focus-visible outline added for keyboard/switch-control navigation
  • Node markers use shape + glyph, not color alone, for colorblind users
  • Home screen now has a working New user / cold-start state, toggleable for review
  • "Why am I seeing this" expands inline instead of linking nowhere
  • Nudge actions are undoable for a short window after tapping
  • Nudge card shows queue position ("1 of 2 today") so users know more exist
  • Onboarding shows time estimate per step and explicit "you can change this later"
  • Onboarding has a learn-more link and autosave confirmation at the end
  • Settings: "Need a break?" pause option sits below full opt-out, as a lower-commitment alternative
  • Settings: Export, Reset, Delete, and Turn off are four distinct, clearly differentiated actions
  • Destructive actions require a confirmation sheet stating exactly what is and isn't lost
  • Theme control added (light / dark / auto) for light sensitivity
  • "See exactly what data this uses" link added next to inferred traits
1. Agent home
Now with a state switcher — try toggling "New user" inside the frame
Original design rationale
R1
Thread motif, not a feed
Cards are connected by a visible line rather than stacked as a generic feed — signals these are one continuous read on the user's day, not unrelated notifications competing for attention.
R2
Briefing leads with pattern, not score
"4 of 6 rings" and "third day this week" describe behavior in context. No grade, no implied judgment.
R3
"Why am I seeing this" on every nudge
A transparency link is always present, not buried in settings — gives the user a way to sanity-check the agent's reasoning in the moment.
R4
"Not now" is equal weight to "Move it"
Dismissal isn't a smaller, greyed-out afterthought — declining a nudge is a first-class action, not friction to push through.
R5
Trend card explicitly avoids a single number
States "no single score here" directly in-product, and gives the Tuesday dip context (offsite) rather than leaving it to read as a failure.
R6
Missed-task reasons are descriptive, not judgmental
"Pushed 3 times," "no time block set" — observations about the task, never framed as something wrong with the person.
R7
Kudos is a single, skippable suggestion
One friend, one moment, with a real "Skip" — not a leaderboard or a pressure-based ask to perform social encouragement.
Accessibility
01
Color was never the only signalFIXED
Each node marker pairs its color with a distinct shape and glyph (✦ briefing, ! nudge, ~ trend, · missed, ♥ kudos) so colorblind users aren't relying on hue to tell cards apart.
02
Text contrast against light backgroundsFIXED
The original light-violet was too close to white (~3.5:1). Body and link text now use a darker violet that clears AA at small sizes; buttons keep the bright violet since it's a solid fill with white text.
03
Tap targets sized for thumbs, not just eyesFIXED
Chips, buttons, and links all carry a minimum touch height even where the visible label is small — a one-line CSS afterthought that's easy to miss until someone tests on an actual phone.
Edge cases & cold start
04
Day-one users have nothing to showFIXED
A populated screen on day one would either be empty and confusing, or worse, fabricated. The New user state explains why each card is blank and what unlocks it, instead of pretending there's a pattern yet.
05
What if there are several nudges, not one?FIXED
"1 of 2 today" sets honest expectations that more exist, rather than the home screen silently picking one and hiding the rest of the queue from the user.
Reversibility & trust
06
"Why am I seeing this" used to go nowhereFIXED
It now expands inline with the actual reasoning ("based on 14 days...") and states the dismissal rule plainly, instead of being a transparency gesture with no follow-through.
07
A mis-tap on "Move it" or "Not now" was permanentFIXED
Both actions now confirm what happened and offer Undo for a few seconds — small, but it's the difference between a tool that tolerates a fat-thumb tap and one that punishes it.
Privacy & social
08
Missed task names are visible on a shared or unlocked screenFLAGGED
"Submit expense report" sitting on a home screen is fine alone, less fine on a lock-screen preview or a screen-share. Worth a "blur sensitive task names" setting, not solved at the mockup layer here.
09
Jordan's streak is shown to someone else without Jordan askedFLAGGED
The kudos card surfaces another person's activity. That needs its own opt-in on Jordan's side — "let friends see your streaks for kudos" — not just the viewer's settings.
2. Agent onboarding
4-step flow, shown here at step 1
Original design rationale
R8
"Skip for now" on every step
Personalization is additive, never a gate — someone can reach the product without answering anything.
R9
No right-answer framing on planning style
Subhead states directly that there's no correct option — pre-empts the instinct to answer how the agent "wants" rather than how they actually behave.
R10
Two separate sliders: frequency and tone
Avoids collapsing "how much" and "how it sounds" into one setting — users may want light-touch + cheerful, or frequent + quiet.
R11
"Nudges are suggestions, not alerts"
Sets expectations before the user has received a single nudge — reframes the relationship from the start.
R12
Editable summary before commit
Final step shows everything back with per-item "Edit," so onboarding ends with confirmation, not a black box.
R13
Privacy stated inline, not just in a policy doc
"None of this is shown to friends unless you choose to share it" appears in the flow itself, where the decision is actually being made.
Expectation-setting
10
No sense of how long this takesFIXED
A bare progress bar doesn't tell anyone if they're 30 seconds or 10 minutes from done. "~1 MIN" per step sets a realistic bar so people don't bail assuming it's longer.
11
"Skip" read as abandoning setup entirelyFIXED
Now reads "Skip for now — you can set this up later," which removes the fear that skipping forfeits personalization permanently.
12
Sliders had no affordance hintFIXED
A flat track can look static rather than draggable. A small "Drag to adjust" caption removes any doubt about whether it's interactive.
Trust & transparency
13
"Light touch" nudges still had no ceiling statedFIXED
The privacy note on step 2 now explicitly mentions a visible daily limit in settings, so "light touch" isn't just a vibe — there's a concrete number behind it.
14
No way to learn more before committingFIXED
Added a "Learn more about how this data is used" link on the confirm step for anyone who wants the longer explanation before finishing, without forcing it on everyone.
15
Did closing the app mid-flow lose progress?FIXED
"Your answers save automatically as you go" on the final step removes the anxiety of a half-finished setup vanishing if someone gets interrupted.
Validation
16
What happens if no focus times are selected?FLAGGED
Step 3 allows zero selections with no guardrail. Needs either a soft minimum-one nudge or an explicit fallback message: "We'll use your activity instead."
17
No calendar-conflict awareness in nudge timingFLAGGED
Onboarding asks when someone's sharpest, but a later nudge ("Move to 8:40am?") could land on a meeting. Worth checking calendar availability before suggesting a time, not just historical pattern.
3. Agent settings
Shown with the "Turn off agent" confirmation sheet open
Original design rationale
R14
"From you" vs. "Inferred" tags
Distinguishes what the user explicitly stated from what the agent observed — corrections feel like fixing a guess, not arguing with the system.
R15
Correction always wins, stated explicitly
The note under "What the agent has learned" commits to this in writing, not just in backend logic — sets a trust expectation the team has to honor.
R16
"Auto-reduce when ignored" is a visible toggle
The fatigue-protection guardrail isn't just backend behavior — it's surfaced as something the user can see is on, building confidence the agent won't nag.
R17
Score display is off by default
Trend view is the default experience; the single productivity number is opt-in, reversing the typical "score-first" pattern of habit apps.
R18
Friend sharing defaults to off, kudos defaults to on
Low-stakes social (suggesting kudos) and higher-stakes social (sharing your own progress/scores) are split into separate toggles with different defaults.
R19
Reset and turn-off are easy to find, not hidden
Leaving personalization is a plain list item, not a multi-step retention flow — consistent with personalization being additive, not a trap.
Destructive-action safety
18
Turn-off and delete were one tap, no confirmationFIXED
Both now open a confirmation sheet that states plainly what's kept and what's lost, with Cancel as the visually equal option — not a quiet "are you sure" buried in a tiny link.
19
"Turn off" and "delete everything" were the same actionFIXED
Split into four distinct rows — Export, Reset, Delete, Turn off — because pausing personalization and permanently erasing history are very different stakes and shouldn't share one button.
20
All-or-nothing nudges: on, or fully offFIXED
"Need a break?" adds Today / 1 week / Until I turn it back on — a lower-commitment middle ground for someone who's overwhelmed this week but doesn't want to lose the setup entirely.
Accessibility & control
21
Toggle state relied on color aloneFIXED
The knob's position (left/right) and background fill both change, so state reads correctly even for users who can't distinguish the on/off color difference.
22
No way to reduce visual intensity for light sensitivityFIXED
Added a Light / Dark / Auto theme control under Appearance — relevant for migraine, light sensitivity, and plain preference, not just an aesthetic option.
Data rights
23
No way to get your data out before leavingFIXED
"Export my agent data" sits above the destructive options — someone shouldn't have to choose between keeping a copy and deleting their data.
24
"What the agent has learned" had no audit trailFLAGGED
"See exactly what data this uses" is linked but not yet designed — needs its own screen showing the specific signals (which days, which tasks) feeding each inferred trait.

Considered, not yet designed

Real problems worth a ticket, but solving them changes scope beyond a screen-level fix — flagging so they don't get lost between this review and a spec.

  • Loading and network-error states for all three screens
  • Push notification permission flow (OS-level, before any nudge can fire)
  • Priority/ordering logic when multiple nudges qualify the same day
  • Calendar-conflict checking before suggesting a specific time
  • Blurring or hiding sensitive task names on lock screen / screen-share
  • Friend-side consent for surfacing their streaks in someone else's kudos card
  • Customizable kudos message instead of a single canned send
  • Minimum-selection handling on the focus-time step
  • Cross-device sync messaging (is this account-level or per-device?)
  • Longer translated strings breaking chip/button layout (i18n)
  • RTL layout mirroring for the thread motif and card alignment
  • Age-appropriate defaults if the product is ever opened to minors

Maps back to a hardened guardrail set

The v1 guardrails (tone, transparency, fatigue protection, scoring risk, opt-in social, additive personalization) still hold. These three are new categories that came out of this pass.

Accessibility

  • Never color-only — shape, glyph, or position always carries state too
  • AA contrast minimum — checked, not assumed, for every text-on-light pairing
  • Real touch targets — sized for a thumb, not just a cursor

Reversibility

  • Undo where it's cheap — nudge actions, in particular
  • Confirm where it's not — anything destructive gets a sheet, not a tap
  • Pause before quit — a lower-commitment option above full opt-out

Honesty about state

  • Cold start is designed, not blank — day-one users get an explanation
  • Queues are visible — "1 of 2" beats silently picking one nudge
  • Transparency links resolve — no link that dead-ends

Data rights

  • Export before delete — never force a choice between the two
  • Granular destructive actions — pause, reset, delete, and turn off are not the same button
